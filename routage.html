Routage HTTP

Le routeur permet de traduire chaque requête HTTP entrante en appel d'action.


Fichier routes

Sur Play, les routes sont définis dans : conf/routes
On définit une route de la manière suivante :
#commentaire
GET    /clients/:id     controllers.Clients.show(id: Long)
Attention à l'appel d'action, le type doit être déclaré après le nom suivi de ':' (comme en Scala)


Routage inverse

Le routage inverse est utilisé pour générer une URL à partir d'un appel Java.
Pour chaque contrôleur, on a un 'reverse controller' dans le 'routes'

Par exemple, si on a
GET    /client/:name      controllers.Clients.get(name: String)
On peut inverser l'URL : controllers.routes.Clients.get("Name")
On peut l'utiliser dans la méthode : redirect()
