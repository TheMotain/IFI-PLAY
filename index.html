TP Play "Kaban"

Le but de ce TP est de créer un tableau de gestion des tâches à l'aide du framework Play

Partie 1 : Affichage d'une vue

  1. Clonez et setup le projet sur votre IDE préféré.
  2. Réaliser un contrôleur BoardController.java (dans le package app.controllers).
    Ce contrôleur va nous servir de point d'entrée des requêtes HTTP.
  3. Dans ce contrôleur, créer une méthode index() qui va renvoyer la réponse ok() avec la vue board.scala.html.
    Pour passer un paramètre à cette vue, il suffit de donner le paramètre à la fonction render() (views.html.board.render(param)).
    Lorsque la va vue sera ajoutée il est possible que eclipse vous remonte une erreur, NE VOUS INQUIETEZ PAS c'est normal.
    Afin que la vue accepte ce paramètre, ajoutez sur la première ligne de la vue board le type des paramètres :
    @(param: TypeParam)
    Faites afficher "hello" à la vue
  4. Ajoutez la route permettant d'accéder à la méthode du contrôleur dans le fichier conf/routes. (chemin : "/")
  5. Regardez le resultat de votre travail sur http://localhost:9000/. N'oubliez pas de lancer sbt : ./sbt run

Bravo, vous avez réussi à afficher une page html dynamique avec PLAY !
Maintenant, nous allons modifier l'affichage de cette page afin d'y afficher un tableau de tâches.

  1. Pour cela, il va falloir aller chercher la liste des tâches à l'aide du service TaskService.java.
    Pour accéder à ce service, il faut l'importer dans le constructeur du contrôleur BoardController à l'aide de l'annotation @Inject.
  2. Une fois injecté, il faut récupérer une List<Task> à l'aide de la fonction getAll() du service.
    Pour le moment, cette fonction renvoie une liste de tâches en dur.
  3. Vous avez maintenant une liste de tâche dans votre méthode index() de votre contrôleur. Il va donc falloir l'envoyer dans votre vue afin de l'afficher.
    Remplacez le paramètre ("hello") de votre vue par votre liste de tâches et ajuster le type du paramètre en haut de votre vue. (typage en Scala)
    Il faut maintenant afficher cette liste grâce à la fonction twirl @for
    Pour le moment, vous pouvez seulement l'afficher, on verra par la suite l'affichage dans la table.

Vous devez maintenant avoir un affichage des tâches existant sur votre page.
Cependant ces données sont codés en dur dans le service. Le but sera maintenant de remplacer ces données en dur par un appel au "repository" et à la base de donnée.

  1. Ouvrez maintenant le fichier TaskRepository.java et créez-y une fonction findAll() qui retournera une liste de tâches.
  2. Dans cette fonction, utilisez les fonctionnalités de EbeanServer afin de retourner la liste des tâches de la base de données (List<Task>).
    Dans un premier temps, cette fonction sera synchrone.
  3. Remplacez maintenant le code dans le service afin d'appeler votre fonction findAll du repository. (Injectez le TaskRepository si nécessaire).
    Vous devriez avoir une autre liste de tâches affiché dans votre page, félicitation si c'est le cas !
  4. Malheuresement, ce code ne respecte pas le principe asynchrone de play. Il ne sera pas très scalable :(
    Pour régler ce problème, il va falloir modifier le code de votre fonction findAll afin de le rendre asynchrone. Pour cela, allez voir le principe de l'asyncrhone
  5. Il va enfin falloir répercuter ce changement dans le reste du code (contrôleur et service).
    Ajustez leur type de retour et dans le contrôleur, modifier votre code afin de gérer l'asynchrone. (voir Contexte d'exécution : HttpExecutionContext pour le contrôleur)

Votre code devra faire exactement la même chose mais en asynchrone... Félicitation ! ;)

Partie 2 : Création une tâche

Dans cette partie, nous allons voir comment créer une tâche et l'enregistrer dans la base de donnée.
Cette partie sera moins guidée que la précédente, n'hésitez pas à aller voir les documentations dans le menu !

  1. Créer une nouvelle vue create.scala.html. N'oubliez pas d'ajouter le @main(title) {...} afin de récupérer le template html de base.
  2. Créer une nouvelle méthode createForm() dans le contrôleur BoardController afin d'afficher cette vue. (ajoutez votre route dans le ficher routes)
  3. Dans cette même méthode, créez un objet Form de type générique Task à l'aide de la factory (FormFactory).
  4. Dans la vue create, créez un formulaire de création de tâche associé à votre objet form à l'aide des helpers twirl.
  5. Votre formulaire va pointer vers une action (une méthode d'un contrôleur).
    Créez donc votre méthode create() dans votre contrôleur et faites pointer le formulaire vers cette méthode.
    N'oubliez pas d'ajouter la route associée dans votre fichier routes !
  6. Pour récupérer les informations entrées dans le formulaire, il va falloir créer un objet Form à l'aide de FormFactory dans la méthode create().
    Ensuite, il va falloir récupérer les valeurs du formulaire dans notre objet form à l'aide de la bonne fonction.
  7. Vous devriez avoir un objet Form dans votre contrôleur comportant les valeurs entrées dans les champs du formulaire.
    Créez maintenant le service et repository associé pour sauvegarder votre Tâche.
    Le service se chargera seulement d'appeler la méthode du répository (insert)
  8. Une fois de retour dans votre contrôleur, il va falloir rediriger l'utilisateur vers la page d'affichage.
  9. Maintenant, gérez les erreurs dans votre formulaire au niveau du contrôleur.

    Si le formulaire est valide, la tâche est créée et l'utilisateur est redirigé sur la liste des tâches.
    Sinon, l'utilisateur est redirigé sur le formulaire avec un message d'erreur. Utilisez flash pour gérer l'erreur !
  10. Si ce n'est pas fait, modifier votre Repository pour effectuer la requête en asynchrone et répercutez ce changement dans votre service et votre contrôleur.
    N'oubliez pas de mettre le contexte courant d'execution HttpExecutionContext . Pour plus d'information, voir le bas de la page Contrôleur
  11. Modifier le lien du bouton dans la vue board afin de rediriger vers la vue create. Vous devriez maintenant être capable d'ajouter une tâche à la liste.

Félicitation, vous avez maintenant un formulaire de création de tâche ! :)

Partie 3 - Edition d'une tâche

Le but de cette partie est d'éditer une tâche afin de changer seulement son status (TODO, IN PROGRESS, READY FOR TEST, DONE).
Pour cela, une édition de la tâche sera nécessaire.

    Dans un premier temps, affichez les tâches en les plaçant dans la bonne colonne du tableau. (code HTML).
    Modifiez l'affichage de vos tâches pour ajouter un lien menant vers un formulaire d'édition.
    Vous devez donc :
    • Créer une méthode edit(id) dans le contrôleur
    • Créer une vue edit.scala.html
    • Ajouter la route
    • Créer un ensemble service + repository afin de récupérer une tâche à partir de son id et d'envoyer la tâche
    • Dans le contrôleur, donner à la vue un formulaire déjà rempli avec la tâche récupérée et un objet Map<String, String> contenant représentant les status (méthode getMap() dans Status).
      Cette map permettra de générer le select des status (voir ci-dessous)
    • Créer un formulaire dans votre vue edit qui pointe vers une autre méthode update(id)
      Votre formulaire contiendra un input pour la description et un select pour le status de la tâche
      Le helper propose un object @select qui génère la balise select automatiquement. Il faut donc donner une map pour qu'il gère les options du select.
      @select(
        taskForm("status"),
        options(status),
        '_label -> "Status", '_default -> "-- Choisir un status --",
        '_showConstraints -> false
      )
    • Créer un ensemble service + repository afin de mettre à jour la tâche
    • Dans la méthode update(id) enregistrez la nouvelle tâche et redirigez vers la liste des tâches en cas de succès ou vers le formulaire d'édition en cas d'echec.

Vous devriez maintenant être capable de modifier une tâche (status) et le status devrait définir dans quelle colonne afficher la tâche. N'oubliez pas de réaliser tout cela en asynchrone ! ;)

Pour aller plus loin

Ce TP touche à sa fin. Si vous le souhaitez, vous pouvez améliorer votre code pour par exemple :

  • Créer une page de login avec un champ "nom" et stocker ce nom en session pour l'afficher en titre de chaque page.
  • Les status sont gérés avec un enum. Changez cela pour enregistrer ces status en base de données!
