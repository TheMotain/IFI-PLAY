L'ORM Ebean

Play contient le plugin Ebean par défaut. Ebean est un ORM destiné à être utilisé avec les langages Java et Kotlin.

Utilisation de Ebean

Pout utiliser les fonctionnalités de Ebean, il va falloir injecter le EbeanServer pour en utiliser ses fonctions.
Pour cela, ajouter à votre classe un attribut private final EbeanServer ebeanServer; et l'annotation @Inject à votre constructeur.
De même, ajoutez l'objet à votre constructeur.

Ebean utilise le concept d'objet Modèle annoté afin de mapper les champs de la table avec les attributs de l'objet.
Par exemple :
@Entity
public class Us extends BaseModel
  @Constraints.Required
  public String name;
  public String description;
}

Select dans la base

Les requêtes SELECT se font en suivant ce modèle :
ebeanServer.find(MonModèle.class).findList();
La fonction find permet de spécifier la table du select (ici la table correspondant au modèle Mon Modèle
La fonction findList correspont à un SELECT *.
Il existe de nombreuses fonction pour filter, trier etc tel que :

  • .where().eq("id", id) pour effectuer un WHERE id=id
  • .orderBy("name") pour effectuer un OrderBy
  • .findOne() pour selectionner que le premier élément retourné

Insertion et Mise à jour

L'insertion d'objet et leur mise à jour se fait simplement à l'aide de la fonction .insert(myModel) ou .update(myModel).
Le binding des champs de l'objet et des champs de la table se fait automatiquement avec les annotations dans le modèle.

Principe de l'asynchrone

Ebean est par défaut synchrone. Ce qui veut dire que la requête va bloquer l'execution du thread.
Pour des raisons de performance lié à Play, il est préférable de modifier ce comportement en asynchrone.
Pour cela, il est possible d'englober notre code dans un bloc asyncrhone à l'aide de la fonction supplyAsync.
Cette fonction prends un lamda en paramètre contenant le code à executer en asyncrhone en premier paramètre et un contexte d'execution en second paramètre.
Le contexte d'execution de ebean est à injecter de la même façon que l'objet EbeanServer et est de type DatabaseExecutionContext

Exemple :
supplyAsync(() -> {
return "coucou";
}, executionContext);

La fonction supplyAsync va retourner un objet de type CompletionStage<T> où T est le type de retour du lambda. (Ici, CompletionStage<String>)
Il faudra alors répercuter ce changement sur les fonction appelantes (dans le service).
