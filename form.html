Formulaire

Lors de la création d'un formulaire, on doit tout d'abord définir l'objet avec ses contraintes : play.data.validation.Contraints
public class Us{
  @Required
  protected String name;

Pour envoyer la vue d'un formulaire, il faut définir un formulaire grâce à play.data.FormFactory (ne pas oublier d'injecter cette dépendance)
Pour former l'objet on utilise formFactory.form(Us.class);
Si on doit récupérer les valeurs existances dans l'objet au niveau de la vue:
usForm.fill(us)

Au niveau de la page html, on peut utiliser une aide au formulaire : helper.

Helper

Il faut importer le module helper au niveau de la vue.

import helper._

On définit un formulaire de cette manière

@form(action = routes.USController.submit(){
   @CSRF.formField
}
Il faut mettre l'annotation
L'action est décrite dans @form(action=.

On peut créer tous les champs avec le @helper.
@form(action = routes.USController.submit(){
   @CSRF.formField
  @inputText(usForm("name")
}

On peut ajouter des attributs à l'input : @inputText(usForm("name"), 'id->"name")
('_label, '_id, '_help, '_showConstraints, '_error et '_showErrors)

Récupération du formulaire au niveau du contrôleur

Pour récupérer un objet du formulaire (pas besoin de le passer en paramètre):
formFactory.form(Us.class).bindFromRequest()

On vérifie ensuite si les données sont correctes :
if (usForm.hasErrors()) {
    return badRequest(views.html.index.render(usForm));
}
On peut également vérifier les données au niveau de twirl :
@if(form.hasGlobalErrors) {
  <p class="error">
    @for(error <- form.globalErrors) {
      <p>@error.format(messages())</p>
    }
  </p>
}
Une fois les valeurs vérifiées, on peut récupérer l'objet de cette manière:
Us us = usForm.bindFromRequest().get();

S'il faut extraire des données lors de la récupération, on a DynamicForm
DynamicForm requestData = formFactory.form().bindFromRequest();
String name = requestData.get("name");
