Moteur de template Twirl

Play est livré avec Twirl qui est basé sur Scala.

Le typage des paramètres est spécifié en utilisant une syntaxe de suffixe
Les types génériques doivent être défini avec des [] et non des <>

Exemples

Le nom d'un fichier doit avoir la forme suivante : nom.scala.html
Lors de la création de ce fichier, une vue : views.html.Application.nom est créée avec la méthode render().
On peut définir les variables utilisées dans la page en haut de la page :
@(story: Us, stories: List[Us])
@(title: String = "Home")

Les paramètres sont insérés dans le contrôleur de cette manière :
views.html.Application.nom.render(us, stories)

Lorsqu'on souhaite faire une instruction dynamique, on utilise @.
Si je veux accéder à ma variable 'story' défini en haut de mon fichier :
@us.name
Si plusieurs appels sont réalisés sur la même ligne, on doit écrire :
@(us.name + us.description)
On peut utiliser des incolades si on a plusieurs instructions.

Il ne faut pas mettre d'espaces entre les mots clés et les paramètres

Pour afficher le caractère '@', il suffit de mettre '@@'

Code
  • Itération - boucle for :
    @for(s <- stories) {
        s.name
    }
  • Condition Si :
    @if(stories.isEmpty()) {
        Vide
    } else {
        stories.size();
    }
  • Commentaires: @* *@
Blocs réutilisables
@display(story: Story) = {
    @story.name
}

Import
@import utils._
@import helper._

Utilisation des modèles en commun
Un modèle 'views/main.scala.html' sera utilisé pour le modèle de présentation général:
@(title: String)(content: Html)
<!DOCTYPE html>
  <html>
    <head>
      <title>@title</title>
    </head>
  <body>
    <section class="content">@content</section>
  </body>
</html>
Dans tous les autres vues, on peut appeler le modèle de cette manière :
@main(title = "Titre") {
    content
}
Les blocs de contenu Html doivent être décrit dans des incolades. On peut également en ajouter
@(title: String)(sidebar: Html)(content: Html)
@main("home"){
  <nav>sidebar</nav>
} {
  <div>content</div>
}

Formulaire
Lors de la création d'un formulaire, on doit tout d'abord définir l'objet avec ses contraintes : play.data.validation.Contraints
public class Us{
  @Required
  protected String name;

Pour envoyer la vue d'un formulaire, il faut définir un formulaire grâce à play.data.FormFactory (ne pas oublier d'injecter cette dépendance)
Pour former l'objet on utilise formFactory.form(Us.class);
Si on doit récupérer les valeurs existances dans l'objet :
usForm.fill(us)

Au niveau de la page html, on peut utiliser une aide au formulaire : helper.
import helper._
@form(action = routes.USController.submit(){}
L'action est décrite dans @form(action=.
On peut créer tous les champs avec le @helper.
@form(action = routes.USController.submit(){
  @inputText(usForm("name")
}

On peut ajouter des attributs à l'input : @inputText(usForm("name"), 'id->"name")
('_label, '_id, '_help, '_showConstraints, '_error et '_showErrors)

Lors de la réception de l'objet au niveau du contrôleur, on vérifie si les données sont correctes :
if (usForm.hasErrors()) {
    return badRequest(views.html.index.render(usForm));
}

On peut également vérifier les données au niveau de twirl :
@if(form.hasGlobalErrors) {
  <p class="error">
    @for(error <- form.globalErrors) {
      <p>@error.format(messages())</p>
    }
  </p>
}

Une fois les valeurs vérifiées, on peut récupérer l'objet :
Us us = usForm.bindFromRequest().get();

S'il faut extraire des données lors de la récupération, on a DynamicForm
DynamicForm requestData = formFactory.form().bindFromRequest();
String name = requestData.get("name");
